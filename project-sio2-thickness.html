SiO₂ Thickness Tool

OceanOptics HR2000+ · thin-film interference · Python

python spectroscopy metrology semiconductor
01 · Overview

Overview

SiO₂ films on silicon wafers scatter white light into iridescent rings — the same physics as a soap bubble. I wrote a Python tool that feeds a reflectance spectrum from an OceanOptics HR2000+ into a fringe-spacing calculation and spits out oxide thickness, no ellipsometer required.

This lives in Prof. Gokirmak's semiconductor fabrication lab. The main use-case is quick, non-destructive QC on thermally-grown or PECVD oxide before the next process step. The device doesn't replace an ellipsometer for thin sub-50 nm films, but for the 100 nm–2 µm range it's fast and good enough for a sanity-check.

02 · How it works

How it works

White light from the HR2000+'s halogen source hits the wafer surface and partially reflects off the air–SiO₂ interface and again off the SiO₂–Si interface. The two reflections interfere constructively and destructively at different wavelengths, producing a sinusoidal ripple in the reflectance spectrum. The fringe period Δλ between adjacent peaks gives thickness directly:

d = (λ₁ × λ₂) / (2 × n × Δλ)

where n ≈ 1.46 for thermal SiO₂ across the visible range. The Python script acquires a spectrum over USB, finds peaks with scipy.signal.find_peaks, averages the fringe spacing across several peak pairs, and reports thickness with an estimated uncertainty based on spectral resolution.

A baseline (reference spectrum off bare silicon or a known reflector) is taken first so the raw counts become a proper reflectance ratio, which removes the lamp's spectral shape and any fiber coupling variation.

03 · Results

Results & measurements

The tool was validated against a thermal oxide sample with a known thickness from the lab's profilometer. Across several measurement spots the tool agreed to within roughly ±5 nm on a ~300 nm film — consistent with the ~1 nm spectral resolution of the HR2000+ at this range.

It fails gracefully below ~80 nm (fewer than two visible fringes in the optical window) and above ~3 µm (fringes too dense to resolve). A report with the full uncertainty budget is in progress.

04 · Notes

Challenges & what's next

The trickiest part was baseline stability: the halogen lamp drifts with temperature, so I added a routine that flags if the reference is stale and re-prompts before a measurement series. Also, the fiber probe angle matters more than expected — even a few degrees off-normal introduces an extra path-length factor. The current version assumes normal incidence and flags the user to re-seat the probe.

Next steps: add a Cauchy dispersion model for n so multi-layer stacks (oxide on nitride, for example) can be handled. Also want to build a small enclosure with a kinematic fiber mount to remove the angle sensitivity entirely.
